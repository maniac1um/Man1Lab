## 4. Architecture Consistency

### Layer Verification

```text
CLI · Console · SDK
        ↓
Platform Facade (Man1Lab)
        ↓
PlatformRuntime → RuntimeSession → SessionWorkspace
        ↓
Business Workflows (Analysis → Discovery → Execution Planning → Execution)
```

| Subsystem | Consistency | Evidence |
|-----------|-------------|----------|
| Runtime | ✅ | `tests/test_runtime_*.py`, `RUNTIME.md` |
| Console | ✅ | Presentation-only; facade delegation; boundary tests |
| Decision Foundation | ✅ | Embedded providers share `providers/embedded/decision_foundation/` |
| Planning | ✅ | `ExecutionPlanningWorkflow` consumes discovery selections |
| Discovery | ✅ | Selection stage + Research Assets |
| Execution Planning | ✅ | Six services, six ports, six embedded providers |
| Provider / Model Registry | ✅ | `LLMManager` via runtime resources |
| Decision artifacts | ✅ | `runtime/session/decision_artifacts.py` uses `importlib` boundary |

### Circular Dependencies

- AST checks: runtime modules do not import `application`
- Console modules do not import forbidden roots (`workflow`, `agents`, `discovery`, `providers`)
- Facade runtime imports limited to allowlisted symbols
