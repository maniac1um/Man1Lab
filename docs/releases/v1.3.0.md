# Man1Lab v1.3.0

**Release date:** 2026-07-14
**Theme:** Execution Runtime and Planning-to-Execution Materialization
**Status:** READY

## Release contract

Man1Lab v1.3.0 can execute a reproduction when Discovery supplies complete, conflict-free, pinned execution evidence. Materialization converts the abstract Planning graph into immutable LocalExecutor task specifications and rejects incomplete or unsupported cases before an `ExecutionRun` is created.

This release does **not** claim arbitrary-paper reproduction.

## Included

- Execution Engine models, scheduler, lifecycle, traces, artifacts, and reports
- Runtime-owned `ExecutionStore` with atomic snapshots and append-only trace persistence
- LocalExecutor and platform/facade/console integration
- Typed repository, dataset, checkpoint, and configuration execution evidence
- Bounded repository and asset preparation with manifests, checksums, safe extraction, and receipts
- One-command application orchestration: Analysis → Discovery → Planning → Materialization → Execution → Report
- Provider-driven pinned DeiT release acceptance
- Cross-process interruption test proving persisted reload, conservative reconciliation, and no duplicate execution of completed tasks

## Supported behavior

- Workspace and pinned Git repositories
- Workspace and HTTPS dataset/checkpoint sources covered by typed evidence
- Existing-file and deterministic-render configuration preparation
- Local subprocess execution with durable logs, artifacts, task attempts, and reports
- Resume of reusable successful tasks after artifact validation
- Safe `BLOCKED`, `UNSUPPORTED`, or `RECONCILIATION_REQUIRED` outcomes

## Known limitations

- Discovery does not guarantee complete execution evidence for arbitrary papers
- Authenticated downloads are not part of the v1.3.0 release contract
- A process-lost LocalExecutor attempt is not automatically redispatched when backend completion cannot be proven
- Docker, remote GPU, cluster, simulation, and robot execution backends are not implemented
- The DeiT acceptance is a bounded offline release workflow; it does not run full ImageNet training or claim scientific metric reproduction
