# Engineering Notes

Informal engineering notes for observations, experiments, and working decisions that do not warrant a full ADR.

## Purpose

Notes capture context that is useful during development but too granular or temporary for ADRs or milestone reviews.

## Format

Each note is a Markdown file. Suggested structure:

```markdown
# Topic Title

**Date:** YYYY-MM-DD

## Observation

What was observed or discovered.

## Decision

What was decided (if any).

## Open Questions

Unresolved items.
```

## Naming Convention

`YYYY-MM-DD-short-topic.md`

Example: `2026-06-28-pdf-extraction-library-choice.md`

## Guidelines

- Notes are informal; they may be superseded by ADRs
- Promote a note to an ADR when the decision becomes architectural
- Do not store secrets, API keys, or credentials
- Keep notes concise
